File structure

We have used the BEM methodology for VSVlog so in result you get a very intuitive file structure. Please click on the folders to discover what's inside.

VSVlog file hierarchy
  • Expand all
  • Collapse all
  • documentation - template documentation
  • template - template files
    • assets - all common and vendor files used
      • fonts - font icons are stored here
      • img - used images
        • common - common images
        • vendor - images used by plugins
      • js - JavaScript files
        • common - common JavaScript functions
        • vendor - JavaScript libraries and plugins
      • less - LESS files
        • common - common less files like mixins, variable declaration, normalize etc.
        • vendor - LESS files required by plugins
    • blocks - all the blocks and their LESS, JS and PHP associated files
      • author - author block
      • box - box block
      • button - button block
      • comment - comment block
      • ... - etc.
    • dist - distributed CSS and JavaScript files
      • dev - development ready scripts used in the dev.html boilerplate
        • blocks.js - general function, bindings and plug-ins initializations.
        • layout.js - JavaScript layout used to wrap the blocks' scripts.
        • vendor.js - JavaScript libraries and plugins concatenated.
      • script.min.js - all the JavaScript files concatenated and minified
      • script.js - all the JavaScript files concatenated
      • style.min.css CSS code concatenated and minified
      • style.css CSS code concatenated
    • dev.html - development boilerplate that include unminified CSS and JavaScript files.
    • index.html - home page
    • about.html - about page
    • article.html - article page
    • ... - etc
  • gulpfile.js - gulp tasks
  • package.json - Node.js dependencies

Getting Started

Using Gulp.js - VSVlog comes with some basic Gulp configuration that should optimize and simplify your workflow. Please follow these steps in order to take full advantage of Gulp's build system.

  • 1

    Install Node.js.

    Go to nodejs.org and download the pre-built installer for your platform. Once it's downloaded, install it.
  • 2

    Open the Node.js command prompt (terminal) and install Gulp globally.

    Within the terminal please run npm install -g gulp. If you are on a Mac, you may need to run it with administrator permissions, so you'll have to run sudo npm install -g gulp instead.
  • 3

    Within the terminal, navigate to your project directory.

    Run cd path/to/your/project/, or you can open the terminal directly in the project's folder. Please note that you should navigate to the folder where the 'gulpfile.js' and the 'package.json' files are located, not to the 'template' folder.
  • 4

    Install the dependencies.

    Once you navigated to the project directory, run npm install and wait until all the required modules will be installed. Node.js is going to create a folder called 'node_modules' and inside that folder all the modules will be installed. Please check that folder for the following modules: browser-sync, gulp, gulp-clean-css, gulp-concat, gulp-less, gulp-rename, gulp-uglify, gulp-watch, gulp-wrap. If any of these are missing, then you'll have to install the missing ones manually. To do so run npm install [missing module], for example if 'browser-sync' is missing, then you'll have to run npm install browser-sync.
  • 5

    Run Gulp

    In the 'gulpfile.js' there are two Gulp tasks predefined - 'dev' task and 'prod' task. Please run gulp dev if you are in development and gulp prod if you finished the development process and want to move the project on the server. The difference between these two is that when you are in development mode gulp is going to watch files for changes and refresh the browser when needed, but in production mode it will concatenate and minify the JavaScript files. If you ran the gulp dev task, please don't close the terminal once all the sub tasks are finished because gulp is still watching for changes and this keeps the local server up and running. To stop the process in the terminal please press CTRL + C.
  • 6

    Start developing

    Since Gulp is watching for file changes, the browser will refresh each time you update any LESS, JavaScript or HTML file.
  • 7

    Edit blocks

    All the block's associated files are in the template/blocks/[block name]/ folder. Gulp watches for changes in those files and will rebuild when any changes are detected.

Without Gulp - VSVlog has two predefined states: development and pre-production

By default all the files are in pre-production mode, that means HTML files include the concatenated versions of the JavaScript and CSS files. If you want to switch to development mode you can use the dev.html boilerplate. The dev.html file includes unminifed and separated JavaScript and CSS files. The minified versions of the CSS and JavaScript files are also available, so you may want to include them into your HTML file if you want to switch to the production state.

Block Structure

BEM stands for Block, Element and Modifier. According to the BEM methodology, a block is a logical and functionally independent page component. In VSVlog every block has its folder inside the "blocks" folder, see below block's file structure.

  • blocks/
    • blockName/
      • blockName.less
      • blockName.js
      • blockName.php

Less Structure

See below the LESS pattern used to style VSVlog.

copy 

.block{
	// Block styles
	text-align: left;
	
	&__element{
		// Element styles
		text-align: center;
	}

	&--modifier{
		// Modifier styles
		text-align: right;
	}
}

Less Variables

You can find below the full list of configurable options available currently in less. In order to update this options, open the "assets/less/common/config.less" and make the necessary changes.

Variable Default Description
@color__primary #e14646 Main theme color.
@color__black #010101 Main black color used for all the texts on the template and for the dark elements.
@color__grey #b5b5b5 Grey color.
@color__grey--light #ededed Light Grey color.
@font__primary 'Roboto', sans-serif Main font family used for the main text.
@font__secondary 'Merriweather', serif Serif font family used for secondary texts.
@icon__primary "simple-line-icons" Main font icons.
@icon__secondary "fontello" Secondary font icons.
@img__common "../assets/img/common/" Path to the common images.
@img__common--background "@{img__common}backgrounds/" Path to the background images.
@img__vendor "../assees/img/vendor/" Path to vendor images.

Preloader

The animated preloader on VSVlog is enabled by default. In case you want to disable it and show the page without it, then just remove the "js-preload-me" class from the <body> tag.

copy 
<!-- Remove the "js-preload-me" class from the body tag -->
<body class="js-preload-me">
	...

	...
</body>

	</a>

Video

You can show a video in two different ways. First one is to redirect the user to the video page (internal link) or to a video hosting provider like YouTube or Vimeo. And the second one is to show the video in a modal. See below how to implement both of them. Please note that the modal supports only YouTube and Vimeo videos.

copy 

<!-- Video Link -->
<a href="https://www.youtube.com/watch?v=ZvPxjhTOEgM">Watch Video on YouTube</a>
<!-- End of Video Link -->

<!-- Video Modal -->
<a href="https://www.youtube.com/watch?v=ZvPxjhTOEgM" class="js-modal-video">Show Video in a Modal</a>
<!-- End of Video Modal -->

Contact form

VSVlog comes with a working contact form with smart data validation. You should configure it first, so please open the 'php/contact.php' file and set the '$myemail' variable to your specific email address where all the messages should be sent. The data will be wrapped by the included email template. Bellow you'll find the form's markup with its default validation attributes. For more information on data validation please visit the plugin's official documentation. The contact form will probably not work on your local server, so you may consider moving your project on your online server when testing it. Also, please make sure your 'php' folder has the 755 permissions set recursively.

copy 

<!-- Form Contact -->

<form action="blocks/form-contact/form-contact.php" class="form-contact js-form-contact">
	<div class="form-contact__modal js-form-contact__modal">
		<h3 class="form-contact__message"> 
			Your message was sent successfully.
			<br>
			Thank you! 
		</h3>
	</div>
	<div class="row">
		<div class="col-md-12">
			<input type="text" class="input" name="full-name" placeholder="Name" data-validation="required">
		</div>
		<div class="col-md-12">
			<input type="text" class="input" name="email" placeholder="Email" data-validation="required email">
		</div>
		<div class="col-md-24">
			<input type="text" class="input" name="subject" placeholder="Subject" data-validation="required">
		</div>
		<div class="col-md-24">
			<textarea name="message" id="" cols="30" rows="10" class="textarea" placeholder="Message" data-validation="required"></textarea>
		</div>
		<div class="col-md-24 right">
			<button class="button ">
				<i class="button__icon icon-paper-plane"></i>Send 
			</button>
		</div>
	</div>
</form>

<!-- End of Form Contact -->
